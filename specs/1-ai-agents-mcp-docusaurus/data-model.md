# Data Model: AI Agents, MCP & Docusaurus Book Project

## Content Entities

### Book
- **id**: Unique identifier for the book
- **title**: "AI Agents, MCP & Docusaurus: A Practical Guide for Students"
- **description**: Comprehensive guide on AI-assisted documentation
- **author**: Generated by AI Documentation Architect
- **chapters**: Array of chapter references
- **status**: draft, in-progress, complete, published

### Chapter
- **id**: Unique identifier for the chapter
- **title**: Chapter title
- **description**: Brief description of chapter content
- **order**: Sequential order in the book
- **sections**: Array of section references
- **status**: draft, in-progress, reviewed, approved
- **word_count**: Number of words in the chapter
- **estimated_reading_time**: In minutes
- **prerequisites**: Array of prerequisite concepts
- **learning_objectives**: Array of learning objectives

### Section
- **id**: Unique identifier for the section
- **title**: Section title
- **content**: Main content in Markdown format
- **type**: concept, tutorial, reference, example, exercise
- **difficulty**: beginner, intermediate, advanced
- **estimated_time**: Time to complete in minutes
- **related_sections**: Array of related section IDs
- **examples**: Array of code/example references

### Example
- **id**: Unique identifier for the example
- **title**: Example title
- **description**: Brief description
- **code**: Code content in appropriate language
- **language**: Programming language or format
- **explanation**: Explanation of the example
- **related_concepts**: Array of concept IDs

### Concept
- **id**: Unique identifier for the concept
- **name**: Concept name
- **definition**: Clear definition
- **examples**: Array of example IDs
- **related_concepts**: Array of related concept IDs
- **difficulty**: beginner, intermediate, advanced

### Skill
- **id**: Unique identifier for the skill
- **name**: Skill name (e.g., "Content Architect", "Style & Consistency Editor")
- **description**: What the skill does
- **parameters**: Input parameters the skill accepts
- **output_format**: Format of the skill's output
- **use_cases**: When to use this skill

### SubAgent
- **id**: Unique identifier for the sub-agent
- **name**: Agent name (e.g., "Research Expert", "Technical Writer")
- **description**: What the agent does
- **capabilities**: List of agent's capabilities
- **input_format**: Format of input the agent expects
- **output_format**: Format of output the agent produces

## Relationships

- A Book contains many Chapters
- A Chapter contains many Sections
- A Section may contain many Examples
- A Section may cover many Concepts
- A Concept may be referenced by many Sections
- An Example may illustrate many Concepts
- A Chapter may require many Skills to complete
- A Chapter may involve many SubAgents

## Content Validation Rules

- Each Chapter must have a unique title within the Book
- Each Chapter must have 1-5 learning objectives
- Each Section must have a clear type (concept, tutorial, etc.)
- Each Example must have a working code sample
- Each Concept must have at least one Example
- Content difficulty must match the target audience
- All external links must be validated
- All code examples must be syntactically correct

## State Transitions

### Chapter States
- draft → in-progress (when writing begins)
- in-progress → reviewed (when initial draft is complete)
- reviewed → approved (after quality validation)
- approved → published (when included in final book)

### Section States
- not-started → in-progress (when section work begins)
- in-progress → complete (when section is finished)
- complete → validated (after quality check)

## Content Structure

The content model follows a hierarchical structure:
- Book (entire textbook)
  - Part (major divisions)
    - Chapter (major sections)
      - Section (subdivisions within chapters)
        - Subsection (further divisions)
          - Content Block (specific pieces of content: text, code, examples, exercises)
