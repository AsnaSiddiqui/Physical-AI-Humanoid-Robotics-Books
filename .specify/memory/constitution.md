## Core Principles:

Technical accuracy with verified information

Clear, beginner-friendly writing style

Consistent structure across all chapters

Practical examples and hands-on tasks

Traceable claims with credible sources

## Key Standards:

Use official documentation as primary references

Inline links or footnotes for all sources

Follow Docusaurus Markdown formatting

Maintain uniform tone, examples, and code style

Use AI tools for support, not blind generation

8–12 chapters, each 800–1500 words

Original writing only (no copy-paste)

Must build and deploy successfully on GitHub Pages

All code and instructions must be tested

## Success Criteria:

All content accurate and verifiable

Clear, consistent, error-free writing

Fully working Docusaurus build

Functional GitHub Pages deployment

No broken links, code, or pages
